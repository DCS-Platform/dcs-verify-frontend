How to check live (since this page doesn't)

Each service can be confirmed independently right now:

curl -sI https://api.dcslabs.ai/v1/explorer/stats   # 200 = API up
curl -sI https://verify.dcslabs.ai/                  # 200 = site up

Verification itself never depends on this page — a credential is checkable directly against the API or entirely offline using the open R+2 standard.

Outage communications

If a service is degraded or down, this is how it's communicated and what each update contains:

  • Where: @DCSAILabs for real-time notices; email to issuer tenants for incidents affecting issuance.
  • Template: [SERVICE] — [investigating | identified | monitoring | resolved] — [what's affected] — [start time UTC] — [next update ETA]
  • Resolution note: a short post-incident summary (cause, impact window, fix) once resolved.

This page is regenerated on each deploy; the build timestamp above reflects that moment, not your current request. No fake "live" indicator is shown.
